커밋 Commit 
프로젝트의 소스코드가 변경되면 이것을 저장소에 저장 즉, 버전을 생성하는 것이다.

풀 Pull
서버(원격 저장소)에 저장된 프로젝트를 로컬(데스크탑)으로 내려받는 것

푸시 Push
로컬 저장소(데스크탑)에서 커밋한 소스를 원격 저장소에 전송
서버에 새로운 버전을 업로드 하는 것

병합 Merge 
버전 관리 중인 프로젝트 라인과 다른 프로젝트의 라인을 합치는 것

패치 Fetch
원격 저장소에 등록된 내용을 로컬에 다운받지 않고 서버에서 확인하는 작업
수정된 내용이 덮어쓰기 되는 것을 방지하기 위해서 사용

브랜치 Branch 
프로젝트 진행 중간에 새로운 기능을 추가하거나 앞에는 같은데 뒤에는 다른 독립적인 개발 라인을 만들 때 쓰는 것
